2FA implemented using better-auth's `twoFactor()` plugin (server) and `twoFactorClient()` (client).

**Files changed:**
- `src/lib/auth.ts` — added `twoFactor()` to plugins, `appName: "Tarteeb AI"`
- `src/lib/auth-client.ts` — added `twoFactorClient()` plugin
- `src/db/schema/index.ts` — now exports from `auth-schema.ts` (generated)
- `src/db/schema/auth-schema.ts` — generated by `npm run ba:generate`; has `twoFactorEnabled` on user and new `two_factor` table
- `src/components/auth/sign-in-form.tsx` — handles `twoFactorRedirect` → navigate to `/auth/two-factor`
- `src/routes/auth/two-factor.tsx` — new: TOTP code input + backup code mode
- `src/routes/auth/route.tsx` — allows `/auth/two-factor` to bypass auth redirect
- `src/routes/account/mfa.tsx` — full enable/disable/QR/backup codes UI
- `src/routes/account/route.tsx` — removed `soon: true` from MFA nav item

**Dependencies added:** `react-qr-code` (pnpm)

**DB migration needed:** Run `pnpm db:generate && pnpm db:migrate` or `pnpm db:push` to apply schema changes (two_factor table + twoFactorEnabled column).

**Why:** User requested full 2FA support via better-auth docs.
**How to apply:** If extending 2FA (OTP via email, trusted devices), refer to better-auth plugin docs. The `authClient.twoFactor.*` methods are the client API.
